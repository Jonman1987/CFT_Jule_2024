Главный файл DataTypesUtility - принимает аргументы строки запуска и передает их в контроллер. Инициализирует контроллер и view.
Статус: Готово.

Контроллер: 
1. Проверяет переданные аргументы строки на валидность.
Статус: Готово. Требует тестирования + рефакторинг кода

2. Инициализирует параметры у модели (тип статистики, имена файлов результатов, пути результатов файлов)
Статус: Готово. Требует тестирования + рефакторинг кода

3. Запускает работу модели
Статус: Готово. Требует тестирования + рефакторинг кода

4. Передает сообщения статусов ошибок во View
Статус: Готово. Требует тестирования + рефакторинг кода

5. Передает сообщения статистики во View
Статус: Готово. Требует тестирования + рефакторинг кода

Модель:
1. Считывает данные с файлов.
Статус: Готово. Требует тестирования + рефакторинг кода

2. Собирает статистику.
Статус: Готово. Требует тестирования + рефакторинг кода

3. Создает файлы
Статус: Готово. Требует тестирования + рефакторинг кода

4. Записывает данные в файлы в виде строк.
Статус: Готово. Требует тестирования + рефакторинг кода

5. Передает данные (цифры) статистики в контроллер в виде строк.
Статус: Готово. Требует тестирования + рефакторинг кода

7. Передает ошибки.
Статус: Готово. Требует тестирования + рефакторинг кода

View:
1. Запуск приложения.
Статус: Готово. Требует тестирования.

2. Отображает сообщение.
Статус: Готово. Требует тестирования.

P.S. Подумать над реализацией Unit тестов
